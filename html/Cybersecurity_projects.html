3D Malware Game

Project Background

Many people are unaware of the dangers of malware. This leads them to unknowingly install malware onto their computer systems and become victims of cyber-attacks. With the rapid rise in malware attacks each year, it is increasingly important to educate the general public on its dangers and how to prevent their devices from getting infected.

The 3D Malware Game is an immersive desktop game. It features many interactive and fun rooms which allow users to move around to learn more about the dangers and threats of malware. In these interactive rooms, players get to enjoy the different game modes as well as learn from informative videos before answering quizzes. By completing the mini-games and quizzes, users will be more informed about the current dangers and threats of malware.

Key Benefits

  • Interactive learning experience – Enables learning in a fun and hands-on manner.
  • Protection from malware via increased awareness – Ability to identify malware if users encounter them in future.

Key Technologies Used

  • Unity Game Engine
  • Visual Studio Code

Potential Enhancements

This application can be enhanced with the following:

  • Wider variety of interactive games and quizzes
  • Better graphics and rendering to enhance immersive experience
  • Support for mobile and VR devices

Arithmetic Circuit Homomorphic Encryption Upgrading RFC7664 to SIDH

Project Background

This project is an extension of an on-going research project on Arithmetic Circuit Homomorphic encryption (ACHE) which was implemented based on Fast Fully Homomorphic Encryption over the Torus (TFHE). Previous project groups have implemented the foundation for single operator calculation and use of Dragonfly Key Exchange (RFC 7664) for authentication, and Abstract Syntax Notation One (ASN.1) X.690 Basic Encoding Rule (BER) as well as expanded the capability to calculate expressions up to two operators.

This project upgrades the current authentication method of RFC 7664 to a Post-Quantum Algorithm called Supersingular Isogeny Diffie-Hellman which makes use of a family of isogenous curves to map from one elliptic curve to another that is a hard problem to crack even for a quantum computer.

Key Benefits

  • Increased speed – Runs functionally faster than previous iterations of the project
  • Increased security – Upgrading to a post-quantum algorithm key pairing further hardens the overall system

Key Technologies Used

  • TFHE
  • Ubuntu 20.04 LTS
  • SIDH

Potential Enhancements

This application can be enhanced with the following:

  • Implementation of a GUI for better user experience
  • Use of floating-point data type for division capability​
  • Use of Domain Name System to contact clients
  • Bug fixes on long exchange time between Output machine and keygen

Automation of Malware Analysis

Project Background

230,000 malwares are discovered every day and this number is expected to grow exponentially. It is challenging for a malware analyst to manually analyse every single malware detected as it is an intensive and tedious process. The analyst might not even be able to perform effective and efficient analysis due to the sheer workload.

Automating Malware Analysis helps to improve classification establishment and better incident response which enhances efficiency, especially in organisations where resources1 are paramount. This project utilises various IT technologies (e.g. AMQP Messaging, RPA Platform, Database Technology) during the entire automated malware analysis process2 to produce a detailed report. This report enables analysts to obtain a general idea of the malware capabilities within a short period of time3, thus allowing them to prioritise resources accordingly.

  • 1 Resources refer to time, manpower, hardware and money
  • 2 Support for Portable Executable file type only
  • 3 Dependent on hardware specifications (~20 minutes)

Key Benefits

  • Reduced human error – Ensures consistency, adherence to processes, compliance and increased security by eliminating mistakes.
  • User-friendly and Scalable – Ease of use and customisable to company’s needs.
  • Efficient – Both static and dynamic analysis are done within a short period of time, saving resources.
  • Resource Prioritisation – Time, manpower, hardware and money can be reallocated to other crucial areas.

Key Technologies Used

  • UiPath, MySQL and RabbitMQ
  • Languages (e.g. Python, Batch, Bash)
  • Analysis Tools (e.g. Cuckoo Sandbox, FLOSS, PPEE)

Potential Enhancements

This application can be enhanced with the following:

  • Better readability and experience for malware analysts
  • Additional support for more file types, analysis tools, and environments

Blockchain Chain of Custody

Project Background

During the investigation of criminal cases, the evidence collected must be handled properly so that it can be used to support the case in court. When evidence is transferred from person to person, there is a chance of the evidence being tampered with. As such, the integrity of the evidence needs to be verified through the chain of custody, which is a record of the movement of the evidence. Details that are tracked include the person who handled the evidence, the date and time it was retrieved or transferred as well as the purpose of retrieval. This process typically involves the physical signing of forms to track the logging in and out process. However, these documents can easily be manipulated.

The Blockchain Chain of Custody system is a tamper-proof solution that traces the movement of the evidence. This digital system records the details of anyone who needs to interact with the evidence. Blockchain technology decentralises the recorded data, making it close to impossible to alter it. In addition, two-factor authentication (2FA) is required for all logins to the system. Hence, this system is a secure solution that maintains the integrity of evidence.

Key Benefits

  • Integrity of evidence – Provides credibility for use in professional environments.
  • Ease of use – Includes user-friendly features, such as search function and report generation.
  • Tamper resistant system – Almost impossible to alter the data.
  • Transparency and accountability – Shows all details of personnel in custody of the evidence.

Key Technologies Used

  • Blockchain
  • Chain-of-custody system
  • Various security measures
  • Distributed ledger system
  • Delegated proof-of-stake

Potential Enhancements

This application can be enhanced with the following:

  • Blockchain mechanism for audit trail
  • Automated adding of evidence
  • Improved advanced search features

Monday Blues

Project Background

This project aims to showcase to the public that commonly used wireless Bluetooth devices such as keyboard and mouse can be exploited via Bluetooth vulnerabilities. An attacker with a Bluetooth dongle sniffer combined with an attack software can wirelessly connect to a victim’s Bluetooth mouse or keyboard. The attacker can then maliciously launch a remote desktop session, activate the webcam and key logger on the victim’s Windows machine to retrieve sensitive information.

To mitigate such an attack, the team created a vulnerable device scanning tool programme that will identify if the connected peripheral devices are vulnerable to any known exploits. It will alert the victim to scan the device followed by suggestions for possible remediation such as updating the firmware on the device.

Key Benefits

  • Alert – Alert victims of the threat of being hacked via Bluetooth to the public by giving them a look into the tools that hackers use to infect victims.
  • Convenient – Checking of vulnerable devices connected to host computers

Key Technologies Used

RPA Integrated Red Teaming Automation

Project Background

Red teaming is a methodology that commonly refers to the practice of testing one’s security systems, aimed to discover system weaknesses and vulnerabilities mainly on operational networks or infrastructure that malicious cyber attackers can exploit. However, these penetration tests need to be done frequently to ensure an adequate security posture.

Finding these weaknesses and vulnerabilities individually is a very manually intensive process. However, many scans are done by hand and are repetitive. They severely handicap the efficiency and productivity of our auditors, preventing them from focusing on more important and intricate issues.

By integrating Robotic Process Automation (RPA), this project aims to eliminate the need for these processes that are done manually by automating them. We have researched and developed scripts and integrated RPA into them, which can substitute the intensive manual penetration testing procedures. RPA allows more time for the auditors to focus on other complex issues. This will aid Red Team auditors, who must conduct frequent tests on multiple IT infrastructures, to produce more accurate and consistent results in shorter time frames.

Key Benefits

  • Ease of use – Intuitive user interface allows users to execute scans easily
  • Efficient – Usage of RPA automates certain processes and reduces downtime
  • Productivity Boost – Lessens the load on auditors, allowing them to work on more complex issues
  • Consistent and Accurate – Scan results mainly focused on consistency and accuracy

Key Technologies Used

  • Python Programming
  • RPA with Python (TagUI)
  • Flask Web Development

Potential Enhancements

This application can be enhanced with the following:

  • More comprehensive dashboard for better data visualization
  • User Management System to track user usage of the software
  • Scheduling Capabilities

The Hacker's Playground

Project Background

The project is designed to complement a core subject “Ethical Hacking and Intrusion Prevention (EHIP)” in the Diploma in Cybersecurity & Digital Forensics. This project provides students with an opportunity to perform cyberattacks on a fictitious organisation and ultimately gain a solid foundational understanding of penetration testing – an authorised simulated cyberattack on a computer system performed to evaluate the security of the system.

The Hacker's Playground serves as a comprehensive, beginner-friendly sandbox environment revolving around a simulated corporation. Its sole purpose is for information security enthusiasts to enhance their hacking skills and practise what they have learned in a fun yet challenging way. The sandbox provides a training ground for learning pen-testing strategies, tools, and industry standards.

Key Benefits

  • Educational – Nurtures individuals in the cybersecurity field

  • Practical – Provides an engaging and educational structure for all participants

Intrusion Defense System

Project Background

Cyber threats have been on the rise in recent years. To combat this, many organisations have resorted to purchasing expensive security solutions. However, Small and Medium-Sized Enterprises (SMEs) may not be able to afford such robust solutions to protect their network.

The TP AMC Network Security Solution is a two-part project to enhance the cybersecurity infrastructure in Temasek Polytechnic’s Advanced Manufacturing Centre (TP AMC). The first part involved rigorous penetration testing to identify specific security concerns within the AMC Lab. Following this, the team created an affordable tool to protect against the identified security flaws in the network. The tool integrates the ntopng software with inexpensive MikroTik network routers to block potential cyber attacks. It also sends alerts to notify users about these incoming attacks. As such, this project provides an effective and low-cost network security solution that can be used by SMEs.

Key Benefits

  • Identifying vulnerabilities
  • Highlighting potential impact on AMC
  • Provide SMEs with a cost-efficient solution.
  • Able to be used as teaching material to facilitate education on cybersecurity engineering.
  • Open-source model to encourage open collaboration.

Key Technologies Used

  • Kali Linux
  • MikroTik
  • ntopng
  • TagUI – RPA
  • Scapy
  • Ubuntu

Potential Enhancements

This application can be enhanced with the following:

  • EtherCAT switch
  • Include contexts such as user, device, application, and more in the alerts.
  • Block off suspicious network ports in addition to the IPv4 address.
  • Enhance the reliability of the RPA program.
