Segregation of Duties

Project Background

Segregation of Duties is an internal control designed to prevent fraud and error by ensuring that two or more individuals are responsible for the separate parts of any tasks, essentially breaking down tasks so that not only one party is solely in control. Trend Micro Cloud One Sales Kit

Project Background

When creating product demonstrations at Trend Micro, sales engineers typically spend around 2 hours to create and test their proof-of-concept demonstrations. This project aims to reduce that time significantly using automation and cloud provisioning tools.

The Cloud One Sales Kit uses various cloud technologies to streamline and automate deployment of technical demonstrations for client presentations. It creates a pre-defined virtual environment on Amazon Web Services that simulates production infrastructure.

Additionally, technical knowledge required to operate this sales kit is kept to a minimum, allowing even the non-technical sales team to showcase all capabilities of Trend Micro’s Cloud One solution professionally.

This project is regularly used by the cloud sales team whenever a product demonstration is required. It has also been used when presenting at conferences and briefings with C-Suite executives.

The Cloud One Sales Kit estimates more than 300 hours saved yearly in productivity.

    Develop Systematic Threats For Smart Industry Systems

    Project Background

    The main goal of our project is to Develop systematic threats for Smart Industry systems. The main purpose of this project is to reduce the exposure these Smart Industry systems face by identifying any zero-day attacks registered in the CVE database.

    Key Features

  • A system architecture based on the Purdue model for ICS Security
  • Hybrid Threat Model Method including STRIDE and Attack Tree threat models
  • A website that displays all threats related to the Smart Industry systems taken from the CVE database
  • Prototype system set up with the Kali Linux operating system to perform simulation attacks using Metasploit.

    • Description

    We started on our project by building a prototype architecture of a Smart Industry system. Then, we proceeded by breaking down and understanding the current Smart Industry systems and using STRIDE and attack tree threat models to identify potential threats and attack vectors. Next, we created a website that allows us to regularly retrieve threats from the CVE database. Last but not least, we used a Metasploit virtual machine to carry out sample attacks with simulation methods.

    Quantum CI Analysis Collector Enhancement

    Project Background

    Quantum's CI analysis collector is a tool that consume scanned outputs from third-party vulnerability assessment(VA) tools.

    The outputs are then transformed into a simple graphical interface for users to view via the Quantum Security Platform. This can be further sorted to match the user's needs like prioritizing remediation efforts.

    The VA tools can scan different aspects of software like images, directories, codes and websites.

    Users would be able to run the analysis collector using the Docker containers that Quantum supplies for each of the officially supported open-sourced VA tools.

    Our Role

  • Integrated majority of the official supported tools into the CI analysis collector.
  • Create unit tests for the code implemented and fix bugs discovered through this process.
  • Create Docker containers for ease of usage so users can operate the tools without installing each of the supported tools.
  • Map/ingest existing mappings for the rules that each tool uses to security frameworks i.e., MITRE ATT&CK.
  • Create the API that is required for developers and users to query from.
