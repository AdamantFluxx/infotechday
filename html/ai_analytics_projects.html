An Automated Model Evaluation Framework

Project Background

After the training process of every machine learning model, there would definitely be a testing and evaluation phase. Currently, in Government Technology Agency (GovTech), there is no structured framework to run the evaluation process of the machine learning models. Besides this, when training a new model or a new version of a model, each AI engineer has to evaluate and track the individual model performance manually.

With this new automated model evaluation framework, AI engineers can have a structured way to evaluate the accuracy and performance of machine learning models. AI engineers can choose to either run the model evaluation framework manually or the automated way. Some of the key features are that evaluation processes on any models can be run with the performance metrics and results logged, stored and exported automatically and monitoring of model degradation with new test data can be done as well.

This project will be used by the AI engineers in GovTech Data Science and Artificial Intelligence Division.

Key Benefits

  • Efficient - Using test automation to replace manual work done by the AI engineers. AI engineers no longer need to manually evaluate and track the results of the models.
  • Convenient - Performance metrics are logged in the same location, making it easy for AI engineers to compare past and newer versions of the models.

Key Technologies Used

  • Flask, HTML, CSS
  • MLflow
  • Pytest
  • Slack API

Potential Enhancements

This project can be enhanced with the following:

  • Adding more performance metrics into the framework
  • Adding charts in exported excel/csv file for visualisations

Chatbots as Personalised Learning Assistants

Project Background

It is not uncommon for students and adult learners to study late into the night, on weekends, during curriculum hours and non-curriculum hours, on campus and even off campus. However, what happens when students have pressing questions, need help and are not sure whom they can turn to for a reliable answer? They can turn to Chatbots – learning companions that can be personalised.

The project aims to ensure that learners can study at their own pace and at any time of the day with a learning assistant to guide them. Scripts have been used to create subject-specific Chatbots on the Google Dialogflow platform. Given the highly automated nature of a Chatbot, any tutor can create one themselves.

Apart from encouraging self-directed learning, these Chatbots are also able to provide personalised learning recommendations based on users’ formative and/or summative performances. Students who are not strong in language skills and have difficulty crafting questions for the Chatbot can opt to use the learning menu to access curated learning materials to better understand concepts.

This project is supported by the Workforce Development Applied Research Fund (WDARF).

Key Benefits

  • Accessible – The Chatbot is available to answer questions 24/7
  • Personalised – Makes recommendations on what learners can review
  • Reliable – Uses Natural Language Processing (NLP) to capture correct intent and provide accurate answers
  • Analytics-enabled – Allows for a companion dashboard for tutors to perform learning analytics

Key Technologies Used

  • Artificial Intelligence (AI) and Natural Language Processing (NLP)
  • Google Cloud platform’s suite of tools for hosting and delivering content
  • Robotic Process Automation (RPA)
  • Levenshtein distance for the recommender system

Potential Enhancements

This application can be enhanced with the following:

  • Supports seamless integration with Learning Management System and MS Teams
  • Adding text and image extractors to augment the Chatbot database curation

Project HIART: Detecting and Resolving Bias in Predictive Models

Project Background

With the current improvements in technology, more companies are starting to add predictive modelling systems as part of their business decision-making processes. However, most of the current predictive models are based on traditional metrics such as accuracy of information provided in the application document or profit margin for the company, and this may lead to situations where certain groups (e.g. low income, women, minority race/ethnicities in less developed countries) do not have fair access to services such as taking up a loan. This is the result of the inherent biases in the dataset.

The objective of the Responsible AI project is to come up with a framework that can be implemented as part of the model building process which will help to recognise and reduce any inherent bias that may inadvertently arise during the process. The framework, with the added fairness metrics, enables the developer to have a clear view of any potential bias or sources of bias in the model building process and then take steps to mitigate it.

The method explored in this project is that of using treatment disparity to mitigate the inherent biases. The final framework uses code nodes implemented in SAS Model Studio to reveal to the user the bias in the dataset so that the user can reduce it through the conscious process of treatment disparity

With this issue of unconscious bias solved, financial institutions can have more confidence to leverage their decision-making process on technology. It will also benefit customers as having a decision-making system that is completely objective allows eligible customers to have fair access to the services they require.

Key Benefits

  • Human Centricity – Prioritises human accessibility over profit
  • Inclusivity – Gives equal oppurtunity to all who need the services regardless of gender, age, race or income
  • Accountability – Addresses biases and mitigates impacts
  • Robustness – Incorporates easily into existing predictive models
  • Transparency – Displays data and information clearly on the dashboard to enable easy access to anyone involved in the decision-making process

Key Technologies Used

  • SAS Viya
  • SAS Visual Machine Learning
  • SAS Model Manager

Potential Enhancements

This application can be enhanced with the following:

  • The model developed can be tested and enriched with more real-life data and synthetic data, where required.
  • The approach can be further extended based on lessons learnt from implementation of various industry frameworks (e.g. MAS F.E.A.T., Japan Social Principles of Human-centric AI, Australia AI Ethics Framework).
  • The lifecycle for model deployment and monitoring can be automated with CI/CD approach.

Drone Vs Drone

Project Background

Unmanned Aerial Vehicles (UAV) are used widely in Singapore to improve safety and efficiency, however, they also poses a potential problem. For example, illegal drone flying could disrupt events. Such incidents would require lots of resources to track and monitor. Furthermore, these resources have limitations in covering large terrains quickly and identifying small objects such as drones from afar.

This solution taps on the use of drones to detect drone activities around Singapore, particularly those that are illegal. This solution makes use of another drone to perform Object Detection with the use of Nvidia technologies. With this solution, the system can detect and intercept illegal drones from afar without a human being physically present. This will enhance the effectiveness as well as increase the reach and range of surveillance activities. The use of drones will also decrease the risks of exposing humans to dangerous substances.

The Object Detection technologies will enable capture and export of highly optimised and accurate Computer Vision Model for edge deployment through locating drones in real time videos.

Key Benefits

  • Convenient – Drones can access a vast distance and difficult-to-reach locations more quickly and effectively than hired personnel
  • Efficient – Drones can preform laborious and routine tasks, thus freeing up man hours for other tasks
  • Security – Drones can be deployed to undertake high-risk tasks and to enhance Singapore Security System through detecting and monitoring illegal drone activities in Singapore

Key Technologies Used

  • Nvidia Transfer Learning Toolkit
  • Nvidia DeepStream
  • Jetson Tx2

Potential Enhancements

This project can be enhanced with the following:

  • Improve model accuracy by training the model with images of different variations
  • Implement a tracking system to track the different drones detected by tagging them with a specific ID
  • Implement GUI for users to easily identify the data/model they want to work on from the AI Platform

Fish Farm Digitalisation

Project Background

Information dissemination on a fish farm is typically done through word of mouth through multiple intermediaries. This may potentially distort the accuracy of the information. A tank of sick fishes may require specific treatments and care instructions, and distorted data may lead to improper treatment and may result in mass mortalities. Besides, a lot of useful information cannot be harnessed through this method. Information that can help to improve communication and processes can be used to optimise the productivity of a fish farm.

Fish Farm Digitalisation aims to utilise these information to improve the traditional business communication processes of a fish farm to minimise errors during information transfer and enhance productivity through better utilisation of information. The centralised website can provide real-time information on the essential distribution of proceedings on the fish farm, the individual fish tank systems, and a feed variety recommender to better utilise the resources while minimising waste.

Key Benefits

  • Efficiency and Efficacy – A centralised platform to display information will eliminate the need for multiple intermediaries during information transfer and minimise errors
  • Convenient – Feed Variety Recommender will suggest feed type upon input of the average bodyweight and species of a fish, which reduces errors and waste.

Key Technologies Used

  • Google Apps Script
  • Google Sites
  • Microsoft Access

Potential Enhancements

This application can be enhanced with the following:

  • Feed Variety Recommender could be implemented on the farm website for all the stakeholders to access easily.

Predictive Maintenance of Fuel Dispenser

Project Background

Changi Airport, as one of the busiest airport in the world, requires all its services and process to be smooth and ready, for its operational efficiency to be maintained. One of such crucial process is refueling planes when they are on the ground. The vehicles are used in the process of refueling the planes and are a vital part of airport operations. Any performance issues of the vehicles will cause delays and cascade downwards to delay in the plane’s departure. Thus, operational performance of the vehicles must be ensured to be optimal throughout.

This project aims to improve the maintenance process for these vehicles, where the current system of maintenance utilizes only preventive and corrective measure. By predicting faults before they occur, helps vehicles to avoid faults that are unforeseen and disruptive to operation as it requires immediate correction. Potentially, it can also lead to cost saving, by reducing number of unnecessary preventive maintenance. By using association rule learning for the machine learning method, we can analyze historical faults, to recognize patterns and trends, and spot plausible causal relationships. This allows us to predict possible upcoming issues with the vehicles before they occur. The analysis insights are then placed into dashboards for operational & analytical use.

Key Benefits

  • Efficient – Using association rule learning to predict possible upcoming faults or parts that required maintenance, instead of routine preventive maintenance which are performed regardless of needs, or corrective maintenance where repairs are done only on occurrence of faults.
  • Convenient – Digitization of vehicle operation, servicing, maintenance, allow data and model to be updated automatically. Enable data-driven decision making for maintenance and procurement of parts and services.

Key Technologies Used

  • Machine Learning (Using KNIME Analytics Platform)
  • Power BI

Potential Enhancements

This project can be enhanced with the following:

  • Additional sensors built onboard the fuel dispenser vehicles.

Video Analytics for the Easy Extraction of Vehicle Attributes

Project Background

Cars are a necessity in a modern, fast-paced society like Singapore. Managing high traffic flow during peak periods can be tricky business, and as such road traffic management is essential to ensure smooth traffic. This project uses deep learning technology embedded in traffic cameras to capture key information in order to better manage traffic. With the identification of vehicle details such as car colour, type and model, it will be easier to track traffic flow count and certain types of vehicle on the road.

Through the use of Yolov4 model and Deepsort algorithm, vehicle types can be identified, and KNN classifier can identify car colours. The Resnet model comes in handy in identifying specific car models. These features allow key vehicle attributes to be extracted automatically from traffic videos which can then be rendered into csv or other video formats for car manufacturers and traffic authorities to use. Car license plate tracking and vehicle tail tracing can also potentially become easier. Beyond the roads, this solution can be implemented in big facility car parking management and for the tracking of vehicles during large-scale events and in smaller districts.

Key Benefits

  • Intelligent System – Vehicle information is easily extracted from traffic videos.
  • Convenient – Automatic processing and analysis of video to capture key vehicle attributes for manufacturer use or traffic management purposes.

Key Technologies Used

  • Deep Learning using Python
  • Yolov4 model trained on COCO dataset
  • Deepsort algorithm used with Yolov4 for vehicle identification
  • KNN classifier trained on color dataset for color identification
  • Resnet model trained on cars dataset for car model identification

Potential Enhancements

This application can be enhanced with the following:

  • Further evaluation of the algorithm for precise color identification.
  • Including a feature for vehicle speed prediction.

Signature Forgery Detection Model

Project Background

The “Know-Your-Customer” business process requires customers to upload their Proof-of-Identity (POI) documents for identity verification and authentication.

This inspired the creation of the Signature Forgery Detection model. The Signature Forgery Detection Model is a binary classification model that is trained using deep learning algorithms to detect forged computer-generated signatures on identification documents.

The model locates and extracts signatures from POI documents and by leveraging the deep learning approach, it predicts whether the document is genuine or fraudulent. Such a model speeds up the validation process of detecting signature manipulation and document forgery within the individual's POI documents effectively and efficiently. This helps the company strengthen its efforts in combating fraud.

Key Benefits

  • Greater Accuracy – Eliminates human error through the use of machine learning algorithms
  • Cost-effective – Automates repetitive tasks of identifying forged signatures to ensure improved efficiency.

Key Technologies Used

  • YOLOv5 Repository (Object Detection Architectures)
  • Python (Programming Language)
  • JupyterLab Notebook (Anaconda)

Potential Enhancements

This application can be enhanced with the following:

  • Elimination of background texture to enable reduction of the noise problem around the signatures for greater success in detection of computer-generated signatures.
  • Training of a new multilingual model to detect the strokes of the signatures of other languages.

Synthetic POI Image Generator

Project Background

During the development of Optical Character Recognition (OCR) projects, a lot of datasets from image documents are needed. One example of such images are Proof of Identity (POI) documents. The process includes the drawing of bounding boxes around the text and the labelling of text and images. The process usually requires much time.

The Synthetic POI Image Generator is an Artificial Intelligence (AI) model based on a Generative Adversarial Network (GAN) structure. This AI model helps users as it semi-automates the process of preparing datasets for AI model training. The Generator successfully generates the synthetic data which replicate the input image and text as closely as possible. In addition, the labels which are generated are stored in a separate text file and will assist in further training of recognition of text. The datasets can be expanded at a very fast rate and once generated, they can be used immediately for the training of the AI model.

Key Benefits

  • Convenient – Allows for greater automated expansion of datasets by 200 – 400 images within one day compared to manual preparation of data.
  • Secure – Reduces the risk of internal data breach through the use of synthetic images to train AI models rather than using classified POI documents.

Key Technologies Used

  • Python
  • TensorFlow
  • Flask
  • HTML5

Potential Enhancements

This application can be enhanced with the following:

  • Allowing support for greater variety of non-alphanumeric characters such as brackets.
  • Allowing support for words longer than 8 characters to ensure more effective training for other AI models.
